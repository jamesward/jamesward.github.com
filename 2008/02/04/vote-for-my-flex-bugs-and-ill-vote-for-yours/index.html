Vote for My Flex Bugs and I'll Vote for Yours!

Feb 4th, 2008 | Comments

A little bug in the Flex Bug DB made it so you could only vote for open bugs - which is kinda silly since deferred bugs are marked as “closed”. Well that bug has been fixed and now it’s time for the Flex Bug Voting Exchange to commence! Here’s how it works - for each bug you vote for of mine (ones I own or just like) - I’ll vote for one of yours! Bugs in bold are worth 2 votes. Just post your bugs you want me to vote for in the comments here. But make sure you vote for mine! Ok, so go and vote for these:

  • Support for mapping Java 5 Enums to strongly-typed AS3 classes

  • set checkPolicyFile on NetStream through VideoDisplay

  • Improving code-completion

  • A Maven2 plugin to integrate Flex into enterprise projects and continuous integration

  • Need a component to render html content (such as from portlets) or at least to provide wsrp/JSR-168

  • Customizing Flash Context Menus

  • No way to create a project which only contains a module

  • Booleans properties should give the user the auto-complete true/false options

  • No option to include the classes from a swc

  • Manifest file is required to be in or under the root source dir -> Need browse button for manifest in the additional compiler args

  • Auto-complete should have suggest-like functionality

  • For effect triggers that have on/off equivalents (like rollOver/rollOut), there should be a single trigger that plays/reverse the effect

  • DataGrid sort performance can be very slow

  • It makes more sense to make the default Link Type for the Flex Framework in library projects to be external - rather than merged into code

  • Should be a 4th Link Type option for libraries - “include all classes”

  • No option to include the classes from a swc

  • Option to obfuscate .swf files to discourage/frustrate decompiling
